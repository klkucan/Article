## GC

###相似点
- 与.NET一样Java的垃圾回收也是无法对非托管资源进行回收的，例如**数据库连接、I/O和物理资源**。
- 在做GC时都会影响性能。
- 当想要尽快对一个对象进行垃圾回收时可以对引用赋值null，显示使用
`System.gc()`会是不错的选择。
- 还是OC的对象设计的好，在对象的内存区域中的有一块引用计数的区域，当release时判断是否引用数为0。

###不同点
- 其实最大的不同就是.NET中采用的generation的算法来进行垃圾回收，而Java则算法很多，而且时机点也不一样，不像.NET就是内存不足时或者手动调用GC方法时。
- .NET中GC时会暂停全部的线程，但是Java中不同的JVM会有不同的策略，有些会暂停全部线程有些却允许线程继续执行，还有的会允许GC在多线程中执行。至于Java中是如何精确记录引用数和嵌套关系的就不得而知了。
- 从书中[1]讲到的HotSpot JVM中有永生代内存这个说法看，应该也是代的概念吧。再次吐槽下**恩恩怨怨**啊😀。
